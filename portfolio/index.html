Cookbook - An application to catalogue recipes; a user can add their personal recipes, view them from a master database, and favorite them. See it on Heroku

  • Rails App with a PostgreSQL database to organize recipes
  • Admin authorization allows for display of recipe statistics such as most common ingredients
  • Uses adapter to scrape a recipe from a Food Network link with Nokogiri
  • Leverages jQuery and AJAX on the front-end to create a cleaner user experience and minimize back-end requests
  • Incorporates fuzzy search to view recipes by ingredient or recipe name

Bring It - A dinner party organizer to invite friends and coordinate a menu without any overlap in what the guests bring. This project was build during week 6 at the Flatiron School. See it on Heroku

  • Rails App with a PostgreSQL database to help organize dinner parties, allowing users to invite friends, create a menu, and ensure guests bring different things
  • Connects with Food2Fork API to suggest recipes based on the menu chosen by the host
  • Leverages advanced ActiveRecord to determine whether a dish has already been claimed by a guest
  • Uses MVC structure to display information and connect domain models

School Domain - An Ember/Rails application to view a course with corresponding teachers and students. This was created one afternoon with classmates as a way to practice building a Rails API and connecting it to an Ember front-end we built.

  • Uses a Rails back-end API to render JSON to the Ember front-end
  • Ember application connects to Rails API with Active Model Adapter
  • Displays teachers through components upon course selection
